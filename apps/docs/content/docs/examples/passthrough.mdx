---
title: Passthrough
description: Advanced OpenClaw keys via hosts.<host>.bots.<botId>.openclaw.
---

Use passthrough when you need an OpenClaw key that is not exposed as a first-class
Clawlets surface.

Where it lives:

- `hosts.<host>.bots.<botId>.openclaw` (raw object merged into the OpenClaw root config)

<Callout type="warning" title="Do not put typed surfaces under openclaw.*">
  Do not set <code>openclaw.channels</code>/<code>openclaw.agents</code>/<code>openclaw.hooks</code>/<code>openclaw.skills</code>/<code>openclaw.plugins</code>.
  Use the typed surfaces on the bot instead.
</Callout>

## Example: set logging defaults

```json
{
  "hosts": {
    "openclaw-fleet-host": {
      "bots": {
        "maren": {
          "openclaw": {
            "logging": {
              "level": "info",
              "redactSensitive": "tools"
            }
          }
        }
      }
    }
  }
}
```

## Example: customize message formatting

```json
{
  "hosts": {
    "openclaw-fleet-host": {
      "bots": {
        "maren": {
          "openclaw": {
            "messages": {
              "responsePrefix": ""
            }
          }
        }
      }
    }
  }
}
```

## How it merges

Passthrough merges into the bot's OpenClaw root config, then Clawlets invariants win
last (gateway bind/port/auth and workspace policy).

See:

- [Typed config surfaces](/configuration/typed-surfaces)
