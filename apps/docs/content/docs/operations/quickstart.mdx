This page is the operator/end-user onboarding flow for dashboard + runner.
Use it when setting up a fresh macOS/Linux workstation.

```bash
clawlets quickstart
```

What it does:

1. Verifies platform + Node runtime requirements.
2. Ensures `nix` exists (installs Determinate Nix when missing).
3. Installs workspace dependencies with `pnpm`.
4. Bootstraps Convex and writes `apps/web/.env.local`.
5. Starts the dashboard UI.

## Non-interactive mode

```bash
clawlets quickstart --no-confirm --ui none --json
```

Useful flags:

- `--install-nix auto|always|never`
- `--skip-nix` (alias of `--install-nix=never`)
- `--nix-bin /path/to/nix` (also persists to `.clawlets/env`)
- `--setup-convex` / `--skip-convex`
- `--convex-dir apps/web`
- `--site-url http://localhost:3000`
- `--ui dev|prod|none`
- `--ui-port 3000`

## Verify

Quickstart is working when:

- The dashboard starts on your local URL (default `http://localhost:3000`)
- `apps/web/.env.local` exists with Convex deployment values
- `.clawlets/env` contains `NIX_BIN`

Next step: open the dashboard setup flow and create/start a runner token.

## If Nix is intentionally skipped

`clawlets quickstart --install-nix=never` exits fast when `nix` is missing.
Install Nix first using the checksum-verified flow in
[Install Nix](/operations/install-nix), then rerun quickstart.

If your shell cannot resolve `nix`, set `NIX_BIN`:

```bash
NIX_BIN=/nix/var/nix/profiles/default/bin/nix
```
